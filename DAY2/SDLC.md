Software Development Lifecycle

Why SDLC?
— To work in an MNC, we need to understand the standard put down by the  software industry to Design, Develop, Test.

Without Design, No development, No delivery without Testing of the developed application and products.

“Case study to understand SDLC”  E - Commerce —> Below is a standard process in a SDLC cycle for any upcoming change. In this case the organization has decided to add a Kid’s clothing catalog which was not there previously.  Planning —> Defining —> Designing —> Building —> Testing —> Deployment

Planning —> There is a high level design, decided by an organization to add the kid’s catalog. Ba’s or PO’s will do requirement gathering, they will take inputs from the customers, if the customers think that launching kid’s catalog, then it will be processed further. Kid’s range also is interested, collecting these information gets into the planning phase.

Defining —> Requirements is drafted in documents for proper brainstorming. Software Requirements are included in such documents.

Designing —> Creating the HLD and LLD.  HLD should be having the core architecture of the applications, like scaling, availability etc.
LLD should be having the core techs and systems which will be used to build the applications.
 
Core DevOps engineer gets involved from this part of the change. Automation should be done and it should be the mindset to automate everything and as possible as it can be. 

Building —>  In this phase, development happens, after you have the LLD and HLD, along with documentations, stories are created and requirements are ready.
Developer will start building the application wrt to the technology being used. After completing the coding, the source code gets pushed to a central location for reference of the other members of the team. Distributed repo system. Pushed to the Git.  Testing —> In this phase, the application from the VCS(Git), is deployed on the server, so the QA team will start testing the software. The qa engineers will test the application end to end.  Deployment —> Final the application gets pushed to the Production server ready to be used by the Customers  How DevOps is Coming into this Picture? — In improving the efficiency of the organization. Basically Fastens the process of the Building, Testing and Deployment and makes sure these things are automated with utmost efficiency
